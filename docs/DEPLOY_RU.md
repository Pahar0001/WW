# Деплой Vela — пошагово (на русском)

Цель: выложить проект на **GitHub** и задеплоить так, чтобы сайт был доступен
другим людям по ссылке.

## Важно понимать архитектуру
Проект состоит из трёх частей, и Netlify **сам по себе не запустит всё**:

```
Netlify (фронтенд Next.js)  ──HTTPS──▶  Render (бэкенд NestJS + база PostgreSQL)
```

- **Netlify** — только фронтенд (то, что видит пользователь).
- **Render** — бэкенд (API) и база данных.
- Они общаются через переменные окружения.

Всё уже подготовлено: есть `render.yaml` (для Render) и `frontend/netlify.toml`
(для Netlify).

---

## Этап 0. Установить git и создать аккаунты
1. Аккаунт на **GitHub** → https://github.com
2. Аккаунт на **Render** → https://render.com (вход через GitHub)
3. Аккаунт на **Netlify** → https://netlify.com (вход через GitHub)

---

## Этап 1. Выложить проект на GitHub
1. На GitHub нажмите **New repository**, назовите, например, `vela`,
   сделайте его **Public** (чтобы другие могли пользоваться), **не** добавляйте
   README/gitignore (они уже есть). Создайте.
2. В терминале, в папке проекта:
```bash
cd ~/Desktop/Repository
git add -A
git commit -m "Vela: ready to deploy"   # если есть несохранённые изменения
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/vela.git
git push -u origin main
```
3. Обновите страницу репозитория на GitHub — файлы должны появиться.

> Секреты не утекут: файл `.env` в `.gitignore` и не попадает в репозиторий.

---

## Этап 2. Бэкенд + база на Render (через Blueprint)
1. В Render: **New +** → **Blueprint**.
2. Выберите свой репозиторий `vela`. Render найдёт `render.yaml` и предложит
   создать **базу `vela-db`** и **сервис `vela-api`**. Нажмите **Apply**.
3. Дождитесь сборки (Docker, несколько минут). В логах сервиса должно появиться
   `Vela API listening...`.
4. Откройте URL сервиса, например `https://vela-api.onrender.com/api/health` —
   должно вернуться `{"status":"ok"}`.
5. Скопируйте адрес API: `https://vela-api.onrender.com`.

> На бесплатном тарифе Render сервис «засыпает» при простое — первый запрос после
> паузы идёт ~30 секунд. Это нормально.

---

## Этап 3. Фронтенд на Netlify
1. В Netlify: **Add new site** → **Import an existing project** → ваш репозиторий.
2. Настройки сборки (часть подставится из `netlify.toml`):
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `.next`
3. Откройте **Site settings → Environment variables**, добавьте:
   | Ключ | Значение |
   |------|----------|
   | `NEXT_PUBLIC_API_URL` | `https://vela-api.onrender.com/api` (ваш адрес API + `/api`) |
4. Нажмите **Deploy**. Получите ссылку вида `https://ваш-сайт.netlify.app`.

---

## Этап 4. Связать фронт и бэк (обязательно)
1. Вернитесь в Render → сервис `vela-api` → **Environment**.
2. Задайте переменную `CORS_ORIGIN` = точный адрес сайта Netlify
   (`https://ваш-сайт.netlify.app`, без слэша в конце).
3. Нажмите **Save** — сервис передеплоится сам.
4. Откройте сайт Netlify — маршрут «Китай» и карта должны загрузиться.

Готово. Ссылкой на сайт Netlify можно делиться с кем угодно.

---

## Частые проблемы
- **На сайте «Пока нет маршрутов»** → неверный `NEXT_PUBLIC_API_URL`, или бэкенд
  «спит»/не поднялся. Проверьте `…/api/health` напрямую.
- **Ошибка CORS в консоли браузера** → `CORS_ORIGIN` на Render не совпадает с
  адресом Netlify (должны совпадать схема и домен, без слэша на конце).
- **Изменили переменную — ничего не поменялось** → нужен **повторный деплой**
  (и на Render, и на Netlify; `NEXT_PUBLIC_*` зашивается при сборке).
- **Загруженные картинки пропали после передеплоя на Render** → на бесплатном
  тарифе диск временный. Для постоянного хранения подключите Render Disk к
  сервису (путь `/app/uploads`) или храните картинки по ссылкам (URL).
- **Карта** работает без ключей (OpenStreetMap) — настраивать ничего не нужно.

---

## Как обновлять сайт после изменений
Просто пушьте в `main` — Render и Netlify пересоберут автоматически:
```bash
git add -A
git commit -m "что изменил"
git push
```

## Альтернатива «всё в одном» (проще)
Если не хочется два сервиса — **Railway** (https://railway.app) умеет поднять весь
`docker-compose` (web + api + db) из этого репозитория сразу. Тогда Netlify не нужен.
Netlify имеет смысл, если важна именно его фронтенд-платформа и CDN.
